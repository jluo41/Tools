🧾  Artifact Contracts — how skills compose
=============================================

  Skills don't call each other in-process — they write plain-text
  files that the next skill reads. The DAG of artifacts IS the
  pipeline. Reading the file list tells you where you are.

─§ 📂 The notes/ canonical set (Stack A · C heavy users) ───

  These are the journal flow's single source of truth. Skills
  in 06–08 ASSUME they exist and are current.

  📂 notes/
    📄 project_truth.md       what the paper currently claims
                              (single source of truth)
    📄 result_summary.md      locked / directional / gapped
                              findings; the claim ledger
    📄 paper_handoff.md       ready-to-draft vs blocked,
                              with artifact pointers
    📄 claim_evidence_map.md  claim → evidence → status
                              (supported / partial / directional)
    📄 decision_log.md        cross-session decisions, so skills
                              don't re-litigate them

  📌 Templates in templates/ (ARIS).
  📌 paper-bootstrap creates them on init.
  📌 Refresh BEFORE manuscript-optimizer / results-section-revision.


─§ 🌐 research-wiki — optional persistent memory ───────────

  When research-wiki/ exists in the project, it becomes the
  cross-skill memory layer:

      /research-lit       auto-ingests papers found
                                │
                                ▼
                        📂 research-wiki/
                                ▲
                                │  reads before ideation
      /idea-creator             │  writes ideas back after
                                │
                                ▼
                        📂 research-wiki/
                                ▲
                                │
      /result-to-claim          │  updates claim status
                                ▼
                        📂 research-wiki/

  Failed ideas → anti-repetition memory. Initialize with
  /research-wiki init.


─§ 📌 The contract is the file ─────────────────────────────

  • If a downstream skill complains "missing X", look for the
    file the upstream skill should have produced.

  • Skills are interchangeable so long as they produce the same
    artifact (paper-write and scientific-writing both write
    paper/main.tex; auto-review-loop and research-review both
    feed NARRATIVE_REPORT.md).

  • This is why Stacks A/B/C/D look so different yet end at
    the same place — different skills, same artifact targets.
