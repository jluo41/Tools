⚙️  DIKW — Inside One  step=task  Iteration
============================================

  Every phase has the SAME shape: ⚙️ step=task, then 🚦 step=gate.
  This file shows what happens for ONE task during step=task.


  🔗  The mandatory sub-skill chain
  ----------------------------------

  🚫 Forbidden shortcut: writing report.md inline, batching tasks
  in one Bash heredoc, or skipping /dikw-context.

       for task in pending_tasks[current_phase]:

         +------------------------------------------+
         |  🔍 Skill( dikw-context                  |
         |            "{phase} {task} {snapshot}" ) |
         +------------------------------------------+
                           |

                           v

           READY   |  BLOCKED|   SKIP |

  ⚡  Two distinct verdict spaces (do not confuse)
  -------------------------------------------------

  +---------------------+------------+-----------------------------------+
  | Skill               | Scope      | Vocabulary                        |
  +---------------------+------------+-----------------------------------+
  | 🔍 /dikw-context    | per task   | ✅ READY / 🚫 BLOCKED / ⏭️ SKIP  |
  | 🚦 /dikw-gate       | per phase  | ✅ approve / 🔄 revise / 🏁 done  |
  +---------------------+------------+-----------------------------------+

  🔍 /dikw-context  =  readiness check  BEFORE the work runs
  🚦 /dikw-gate     =  quality check    AFTER  the phase is done


  📦  Artifact contract — task is "complete" iff
  -----------------------------------------------

  +------+------------------------------------------------------------+
  | 📊 D |  🐍 analysis.py  exists  AND  was executed              |
  |      |  📄 report.md    non-empty, concise, max ~1000 words    |
  +------+------------------------------------------------------------+
  | 📈 I |  🐍 analysis.py  exists  AND  was executed              |
  |      |  📄 report.md    non-empty, concise, max ~1000 words    |
  +------+------------------------------------------------------------+
  | 🧠 K |  📄 report.md    non-empty, concise, max ~1000 words    |
  |      |  (reasoning only, no .py)                               |
  +------+------------------------------------------------------------+
  | 🎯 W |  📄 report.md    non-empty, concise, max ~1000 words    |
  |      |  (reasoning only, no .py)                               |
  +------+------------------------------------------------------------+

  ⚠️  Inlining a Python script in a Bash heredoc instead of saving
  🐍 analysis.py is a contract failure — the task folder is not
  re-runnable.


  📂  Folder-naming for a task
  -----------------------------

      {L}{NN}-{slug}/

      L    = D | I | K | W           (phase letter)
      NN   = 01, 02, ...             (next available within that level)
      slug = task_name from the plan
