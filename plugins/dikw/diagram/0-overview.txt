🗺️  DIKW Plugin — 3-Tier Skill Map
===================================

  How the pieces fit. Two commands a human normally types (Tier 1).
  Everything else is reachable but invoked by the orchestrator.

─§ 📌 Two cross-cutting truths ─────────────────────────────

  📌 Two cross-cutting truths
  ---------------------------

  1.  💾 State lives ONLY on disk (DIKW_STATE.json + plan files +
      gate files + insights/).  The orchestrator re-reads it on
      every loop turn — so resume-from-anywhere is free.

  2.  🚦 Routing vocabulary is exactly THREE outcomes:
        ✅ approve  |  🔄 revise [feedback]  |  🏁 done
      No others.  Every back-edge routes to plan.
      🏁 done is restricted to G-K, G-W, G-report (skipping K/W
      reasoning would produce an incomplete final report).
