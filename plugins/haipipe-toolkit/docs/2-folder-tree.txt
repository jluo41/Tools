📦 haipipe-toolkit/ — Skill Folder Layout
==========================================

Numeric prefixes follow the data-flow order (data → nn → end), then
cross-cutting (project), then per-subject (subject). Folder names are
pure organization — only the `name:` field in each SKILL.md frontmatter
identifies the skill. Claude Code walks the tree at any depth.

📌 Convention recap
--------------------
  🎯 umbrella    = orchestrator; parses intent, dispatches via Skill()
  🔧 specialist  = executor; reads its own ref/concepts.md + umbrella's
                   shared fn/*.md; emits structured return tail
  📁 ref/        = reference content (concepts, overviews)
  📁 fn/         = procedure docs (shared across specialists in a group)
