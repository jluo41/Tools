🔁 haipipe Pipeline — 7 Stores, 6 Build Stages
================================================

Project-wide build flow (left → right). Each Store is a folder under
`_WorkSpace/`. Stage 0 is INPUT (raw dumps) — not built by haipipe;
the other 6 stages are owned by an umbrella as labelled below.


Build flow — what builds what
------------------------------

  📥 INPUT                 ──── 🎯 /haipipe-data ────                 ──── 🎯 /haipipe-nn ────             ──── 🎯 /haipipe-end ────

  +----------------+   +---------------+   +-------------+   +---------------+   +----------------+   +-------------------------+   +-------------------+
  | 0-RawDataStore |──>| 1-SourceStore |──>| 2-RecStore  |──>| 3-CaseStore   |──>| 4-AIDataStore  |──>| 5-ModelInstanceStore    |──>| 6-EndpointStore   |
  | dataset dumps  |   | typed frames  |   | records     |   | cohort cases  |   | train tensors  |   | trained weights         |   | deployable pkg    |
  +----------------+   +---------------+   +-------------+   +---------------+   +----------------+   +-------------------------+   +-------------------+
        S0                  S1                  S2                  S3                  S4                  S5                          S6
       (input)              │                   │                   │                   │                   │                            │
                            v                   v                   v                   v                   v                            v
                       SourceFn /          RecordFn /           CaseFn               TfmFn /             Algorithm                  Endpoint_Set
                       HumanFn             TriggerFn                                 SplitFn             + Tuner                    artifact
                                                                                                         + Instance                 + 4 deploy
                                                                                                         + ModelSet                  targets


Stage → specialist mapping
---------------------------

  +-------+----------------+--------------------------------+----------------------------+
  | Stage | Store          | Builds (functions)             | Skill                      |
  +-------+----------------+--------------------------------+----------------------------+
  | S0    | RawDataStore   | (input — not built by haipipe) | n/a                        |
  | S1    | SourceStore    | SourceFn, HumanFn              | haipipe-data-source        |
  | S2    | RecStore       | RecordFn, TriggerFn            | haipipe-data-record        |
  | S3    | CaseStore      | CaseFn                         | haipipe-data-case          |
  | S4    | AIDataStore    | TfmFn, SplitFn                 | haipipe-data-aidata        |
  | S5    | ModelInstance  | Algorithm/Tuner/Instance/Set   | haipipe-nn-{algo,...}      |
  | S6    | EndpointStore  | Endpoint_Set + deploy          | haipipe-end-endpointset    |
  |       |                |                                | + per-Fn-type:             |
  |       |                |                                |   -{meta,trig,post,        |
  |       |                |                                |     src2input,input2src}   |
  |       |                |                                | + per-deploy:              |
  |       |                |                                |   -deploy-{sagemaker,      |
  |       |                |                                |     databricks,local,      |
  |       |                |                                |     mlflow}                |
  +-------+----------------+--------------------------------+----------------------------+


Per-subject slice (inference time — see also 5-subject-vs-project.txt)
-----------------------------------------------------------------------

   _WorkSpace/A-User-Store/UserGroup-{dataset}/Subject-{id}/
   ├── 📂 0-RawDataStore/    ← subject's raw rows / files (FLAT)
   ├── 📂 1-SourceStore/     ← subject's source frames    (FLAT, no @iXnY)
   ├── 📂 2-RecStore/        ← subject's record(s)        (FLAT, no @iXnY)
   └── 📄 manifest.yaml      ← provenance: source_set, rec_set, partition

                                  │
                                  v  serves a deployed endpoint with
                                     this subject's record
                                  │
                                  v
                          🎯 /haipipe-subject

   Stages 3-6 NEVER appear per-subject — at inference, the subject
   simply CALLS the deployed endpoint built from project-wide 6-Endpoint.


Build verbs that map to each stage (haipipe-data verb map)
-----------------------------------------------------------

  +------------------+--------------------------------------------+
  | Verb             | What it does                               |
  +------------------+--------------------------------------------+
  | design-chef      | scaffold a new fn (the "recipe")           |
  | design-kitchen   | modify the surrounding pipeline ("kitchen")|
  | cook / run       | execute the fn, materialize the store      |
  | load             | inspect the store output                   |
  | review           | audit fn files / outputs                   |
  | dashboard        | per-stage status                           |
  | explain          | cross-stage Q&A (umbrella inline)          |
  +------------------+--------------------------------------------+


Compact form
------------

  📥 0-Raw → 📊 1-Source → 📝 2-Rec → 📂 3-Case → 🎲 4-AIData → 🧠 5-Model → 🚀 6-Endpoint
   input    └────────── /haipipe-data (S1-S4) ──────────┘  └ /haipipe-nn ┘ └ /haipipe-end ┘
                                                                              (artifact +
                                                                               4 targets)
   subject ─ /haipipe-subject reuses S0-S2 only (filtered view of global stores)
