🔁  Nested cycles — narrative ⊃ probe ⊃ task
=============================================

  The system runs as three nested cycles. Each WHIPS the level below and
  PRODUCES one thing. (The L-labels map each cycle to existing machinery.)

  Prose: ../../HANDOFF.md (LATEST MOVEMENT)
         ../../skills/D_probe/haipipe-probe-loop/SKILL.md · probe-cycle steps → 06


─§ 🪆 The nesting ──────────────────────────────────────────

┌─ 📖 narrative-cycle ── 1 narrative turn; whips N probe cycles    [N_narrative ✅ scope A]
│     立目标 → plan → 读 🧠insights → 🔧 whip probe cycles → 回读 → 🔥 ignite?
│     ignite=ready → 🥢 application 变现 (📰/📊/💬/🖥️)
│
│  ┌─ 🔧 probe-cycle ── 1 probe → N task-cycles → 1 insight        [steps → 06]
│  │     ① design → ② bridge → ③ run×N → ④ result → ⑤ verdict → ⑥ insight
│  │     (probe-loop iterates ①–⑤ in "rounds" until ✅; ⑥ files on convergence)
│  │
│  │  ┌─ ✋ task-cycle ── 1 task: config → run → result            [C_task]
│  │  │     author → 🚦 Run Script Reviewer → bash run.sh → 🚦 Run Result Auditor → result
│  │  └──────────────────────────────────────────────────────────────
│  └─────────────────────────────────────────────────────────────────
└────────────────────────────────────────────────────────────────────

  📌 each cycle is the callee of the one above: 📖 narrative whips 🔧 probe cycles;
     a 🔧 probe cycle whips ✋ task cycles. 🧠 E only FILES (⑥) — it never drives.
