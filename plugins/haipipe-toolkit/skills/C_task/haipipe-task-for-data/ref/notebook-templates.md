Notebook Template → Task Instance Pattern
==========================================

Each task `.py` is an **exact copy** of a template from
`code/scripts/haistepnb/`. The template is copied as-is to the task
folder and renamed. CONFIG is overridden at runtime by papermill
(`-p CONFIG <path>`) or `run.sh`, NOT by editing the `.py` file.


Templates (ABC naming)
-----------------------

```
code/scripts/haistepnb/
  a1_source_nb.py      Stage A1 — SourceSet from raw csv/parquet
  a2_record_nb.py      Stage A2 — RecordSet from SourceSet (multi-partition)
  a3_case_nb.py        Stage A3 — CaseSet from RecordSet (multi-partition, parallel)
  a4_aidata_nb.py      Stage A4 — AIDataSet from CaseSets (multi-partition merge)
  b_model_nb.py        Stage B  — ModelInstance (train + examples + validate)
  c_endpoint_nb.py     Stage C  — Endpoint_Set (package + test payloads)
```


Instantiation
--------------

```
Template                              Task instance
────────────────────────────────      ──────────────────────────────────────────
haistepnb/a1_source_nb.py        →   A01_.../01_source_mimiciv/1_source_mimiciv31.py
haistepnb/a2_record_nb.py        →   A01_.../02_record_mimiciv/2_record_mimiciv31.py
haistepnb/a3_case_nb.py          →   A01_.../03_case_mimiciv_mortality/3_case_mimiciv31_mortality.py
haistepnb/a4_aidata_nb.py        →   A01_.../04_aidata_mimiciv_mortality/4_aidata_mimiciv31_mortality.py
haistepnb/b_model_nb.py          →   B01_.../01_xgb_mortality/1_train_xgb_mortality.py
haistepnb/c_endpoint_nb.py       →   C01_.../01_endpoint_mortality/1_endpoint_mortality.py
```

To instantiate:
1. Copy the template to the task folder (exact copy — do NOT edit CONFIG)
2. Rename to `{NN}_{task_name}.py`
3. Create `configs/run_{task_name}.yaml` with project-specific settings
4. Create `runs/run_{task_name}.sh` that passes CONFIG via papermill

CONFIG is injected at runtime:
```bash
papermill template.ipynb output.ipynb -p CONFIG examples/.../configs/run_xgb_mortality.yaml
```


Task Group → Template Mapping
-------------------------------

```
Group letter   Template           Pipeline stage
────────────   ──────────         ──────────────
A01_*          aa/ab/ac/ad_*_nb   Data pipeline (Source→Record→Case→AIData)
B01_*          b_model_nb         Model training (with ExampleConfig for examples)
C01_*          c_endpoint_nb      Endpoint packaging (Endpoint_Pipeline + Src2InputFn → payload.json)
```


Source vs Output
-----------------

```
{task}/{NN}_{task_name}.py         ← SOURCE (exact template copy; commit this)
{task}/{NN}_{task_name}.ipynb      ← OUTPUT (auto-generated by convert_to_notebooks.py at runtime)
{task}/notebooks/<RUN>.ipynb       ← OUTPUT (papermill execution record)
```

The `.ipynb` at the task root is a template notebook generated by `run.sh`
step 1 (`convert_to_notebooks.py`). It is NOT source — do not edit it.
The notebook under `notebooks/` is the papermill execution record with
cell outputs.


Partition parameters
---------------------

Stage A2-A4 templates include these papermill parameters:

```python
# %% [parameters]
CONFIG         = "..."
REPO_ROOT      = ""
NUM_PARTITIONS = 0      # 0 = use config / auto-discover
PARTITION_INDEX = ""    # "" = all; int = one partition (1-based)
NUM_WORKERS    = 1      # >1 for parallel (Stage A3 only)
```

Inject via papermill:
```bash
papermill template.ipynb output.ipynb -p CONFIG ... -p NUM_PARTITIONS 20
```


CLI scripts (alternative to notebook)
--------------------------------------

Each stage also has a CLI script with full partition support:

```
code/scripts/haistepcli/
  source.py     python code/scripts/haistepcli/source.py --config <config>
  record.py     python code/scripts/haistepcli/record.py --config <config> --num-partitions 20
  case.py       python code/scripts/haistepcli/case.py   --config <config> --num-workers 4
  aidata.py     python code/scripts/haistepcli/aidata.py --config <config>
  model.py      python code/scripts/haistepcli/model.py  --config <config>
  endpoint.py   python code/scripts/haistepcli/endpoint.py --config <config>
```

The CLI and notebook use the same underlying pipeline code. Choose CLI
for scripted pipelines; notebook for interactive inspection and Databricks.
