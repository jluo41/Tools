─§ 2. What Plan / Build / Execute / Report MEAN ───────────────────

Each stage answers ONE question and produces ONE artifact set.
All 4 stages operate on the SAME task folder's workflow/ directory.

📍 Stage 1: PLAN — "What should this task do?"
──────────────────────────────────────────────────
  I: existing .py script + type specialist's workflow-plan-sample.yaml
  P: 🔨 creator reads code, maps phases, writes IPO-compliant YAML
     🔍 reviewer checks schema compliance + phase accuracy
  O: workflow/plan.yaml          (task-level: Run/Gate1/Gate2)
     workflow/plan-script-*.yaml (script-level: type-specific phases)

     The plan is the CONTRACT. Everything else refers back to it.

🔨 Stage 2: BUILD — "Is the code ready to run?"
──────────────────────────────────────────────────
  I: the plan (from Stage 1) + existing code (may be messy)
  P: 🔨 creator writes/fixes .py, configs, runs/, structure
     🔍 reviewer checks for intent-vs-implementation bugs (Gate 1)
  O: 01_band4.py               (complete, config-driven, # %% cells)
     configs/run_band4.yaml     (frozen params with _meta)
     runs/run_band4.sh          (papermill wrapper + pre-flight gate)
     notebooks/                 (dir ready for runtime output)
     CODE_REVIEW.md             (Gate 1 verdict)

     After Build, the task folder is COMPLETE and READY TO RUN.

▶️ Stage 3: EXECUTE — "Run it and wait."
──────────────────────────────────────────────────
  I: the ready task folder (from Stage 2)
  P: bash runs/run_band4.sh     (no creator — just press the button)
     🔍 reviewer audits outputs vs plan contract (Gate 2)
  O: results/run_band4/         (metrics.json, runtime.yaml, comparison.md)
     notebooks/run_band4.ipynb  (papermill execution record)
     RUN_AUDIT.md               (Gate 2 verdict)

     Execute is DUMB — no thinking, no modification, just run + audit.
     Optional (autoExecute=false by default).

📋 Stage 4: REPORT — "What actually happened?"
──────────────────────────────────────────────────
  I: the plan (from Stage 1) + results (from Stage 3) + gate verdicts
  P: 🔨 creator mirrors plan structure with ✅/⏭️/❌ per step
     🔍 reviewer checks accuracy vs actual files on disk
  O: workflow/report.yaml              (task-level, mirrors plan.yaml)
     workflow/report-script-*.yaml     (script-level, mirrors plan-script)

     Plan = intent. Report = evidence. Same shape, diffed to find gaps.
     Optional (autoReport=false by default).
