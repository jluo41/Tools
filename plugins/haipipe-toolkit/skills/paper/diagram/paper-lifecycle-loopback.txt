Diagnosis-driven loopbacks
--------------------------

                        +--------------------+
                        |    9 Review Gate   |
                        +--------------------+
                          |   |   |   |   |
       local prose/cite --+   |   |   |   +-- whole idea no longer viable
                              |   |   |       -> 0 Seed
                              |   |   |
                              |   |   +------ public story failed
                              |   |           -> 2 Paper Pitch
                              |   |
                              |   +---------- claim/evidence mismatch
                              |               -> 3 Narrative
                              |               -> discovery/task/probe if evidence missing
                              |
                              +-------------- section/figure/argument shape wrong
                                              -> 4 Architecture or 5 Plan

       local prose/cite/number/format
       -> 8 Edit Cycle


Layer map
---------

  Story layer
    0 Seed
    2 Paper Pitch

  Evidence contract layer
    3 Evidence-Backed Narrative

  Paper shape layer
    4 Architecture / Minimap
    5 Paper Plan
    6 Build Skeleton

  Text realization layer
    7 Write Draft
    8 Edit Cycle

  External gate layer
    9 Review Gate
    10 Submit
    11 Respond / Revise
    12 Present


Rule
----

  A paper moves forward through pitch, narrative, architecture, plan, draft,
  and review; feedback loops back to the earliest layer that explains the
  failure.

  Loopback is chosen by diagnosis, not by chronology.
