When doing editing scaffold work, don't compile the PDF first. We need to first make sure we have the editing plan being ready. The user will also provide feedback on the outline before it goes to tex.

Correct workflow order:
1. Create/update the editing scaffold (outline .md)
2. Present outline for user review
3. Iterate with user feedback paragraph by paragraph
4. Once outline is settled and confirmed, THEN sync to tex
5. THEN compile PDF

The agent jumped from creating the consolidated §3 outline directly to rewriting 03_theoretical_framework.tex and compiling, skipping user review of the outline entirely.
