# Stage 00: Topic Appears

## What This Scene Does

Stage 00 starts when someone says: "I think there may be a paper here."

The job is not to write. The job is to classify the topic:

- **Project-only**: keep exploring inside the project lifecycle.
- **Paper prospectus**: create an early paper container with a thin direction.
- **Paper seed**: enough evidence exists to start the paper lifecycle proper.

## Problem It Solves

It prevents a raw idea from becoming a premature manuscript. A topic can have a
paper-shaped direction before it has a claim, evidence, venue, display contract,
or draft.

## Artifacts It Reads

Project-side:

- project seed or project README
- narrative notes
- discovery notes
- task/probe plans
- early insights, if any

Paper-side:

- usually none yet

## Files It Changes

If the topic stays project-only, no paper files are required.

If the topic becomes a paper prospectus, the next stage creates:

```text
lifecycle/stage00_topic-appears/current.md
```

This file is the paper-shaped discovery constraint. It is not an abstract,
outline, pitch, or paper plan.

## When It Moves Forward

Move to Stage 01 when the topic has enough direction to justify a concrete paper
container.

## When It Loops Back

Stay in the project lifecycle when:

- the phenomenon is too vague;
- there is no candidate outcome or empirical object;
- the topic is still just curiosity;
- the right next move is discovery, probe, task, or narrative work.

## Relation To Project Lifecycle

Stage 00 can be triggered by project narrative, a discovery, a task result, a
probe idea, or an author hunch. It does not own evidence production.

The preferred handoff is:

```text
project -> paper prospectus -> project narrative -> discover/probe/task/insight
```
