Illuminate + Elicit Protocol
=============================

Producing the stage artifact is not enough. Before and while drafting, the
agent must **explain** and **illuminate** the stage content to the user, and
**ask** for the user's judgment. The paper is the user's; the agent drafts on
their behalf, not decides for them. This is a participatory / Socratic
requirement that every stage skill must apply.


The Protocol (Step 0 of every stage)
-------------------------------------

1. **PRESENT** the current state in plain language: "Here is what <stage>
   currently says / could say."

2. **IDENTIFY** 2-3 taste-bearing decisions -- framing, emphasis, scope,
   ordering, what to foreground vs background. State your recommendation and
   why.

3. **ASK** the user for their take on those decisions. Wait for input.

4. **DRAFT** with the user's preferences folded in.

5. **RE-WALK** (for an already-drafted stage): surface what is ALREADY there
   and ask "keep / change / reframe?" per taste-bearing element. Do NOT
   silently overwrite prior choices.


Per-stage taste-bearing decisions (examples, not exhaustive)
-------------------------------------------------------------

- **seed** -- Which question shape? Which kill criteria matter most? Which
  evidence path to pursue first?
- **pitch** -- Which hook method (paradox / vivid scene / surprising fact / gap
  / question)? Which audience emphasis? What is the "one-minute repeat"
  sentence?
- **claims** -- Which claim is PRIMARY for this venue? Which claims to
  foreground vs relegate to supporting? What frame (clinical / method / policy)?
- **narrative** -- Which 5-act arc variant? How to order the story beats? What
  emphasis for limitations?
- **display** -- Which display is the hero figure? Which displays are main vs
  supplement? What visual form (table vs figure vs forest vs flow)?
- **minimap** -- How to allocate page budget across sections? Which paragraphs
  carry the most argumentative weight?


Autonomy boundary
------------------

- **Taste-bearing choices = PAUSE to elicit** (the choices listed above).
- **Mechanical work = autonomous** (formatting, compiling, sentence-indexing,
  file scaffolding).
- **Rule of thumb**: if removing the choice would change what the paper ARGUES
  or how it READS, it is taste-bearing.


Compile after every edit
-------------------------

After the stage artifact is drafted or edited, compile the PDF (per
ref/tex-quality.md). The compiled PDF is what the user reads to internalize the
stage. Without a fresh PDF, there is nothing to illuminate.
