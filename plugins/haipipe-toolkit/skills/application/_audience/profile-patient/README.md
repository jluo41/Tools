# Audience Profile: Patient

## Tone
Warm, plain language. Second-person ("you can..."). Empathetic
but not condescending. Actionable.

## Constraints
- Length: ≤ 200 words
- Jargon: avoid all medical/technical jargon
- Reading level: 6th grade
- Citations: NO K-id visible in body; cited_K in frontmatter only

## Lifecycle mappings

### → Claims
Claims must be translatable to plain benefit statements.
"HTE shows 8pp improvement" → "this can help you refill on time."
Clinical precision not required; directional accuracy is.

### → Design
Lead with benefit, not action. Include a clear call-to-action.
Personalize with patient name if available. No URLs longer than
30 characters (use short links).

### → Variants
Standard set: one primary SMS + one follow-up if no response.
Keep variant count low — patients don't need options, they need
clarity.

### → Draft
Imitate the style in exemplars/. Warm opener, 1-2 benefit
sentences, clear CTA, friendly close.
