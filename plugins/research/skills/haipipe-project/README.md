haipipe-project
===============

Skill for creating, reviewing, and summarizing haipipe research projects.
Covers both the experiment side (examples/) and the code side (code/ + code-dev/)
in one unified flow.

---

Commands
--------

  /haipipe-project new
      Start a new project from scratch. Asks a few questions (project ID,
      dataset, stages, new code needed?), then scaffolds the full folder
      structure under examples/ and optionally creates Track A stubs in
      code-dev/ and code/hainn/. Checks code/INDEX.md first to suggest
      reusing existing Fns or models across projects.

  /haipipe-project review
  /haipipe-project review examples/ProjC-Model-WeightPredict
      Inspect an existing project AND generate any missing standard docs.
      Auto-detects the current project from git status if no path is given.

      What it checks (read-only):
        - Project naming convention
        - Four-part folder structure (cc-archive, config, scripts, docs)
        - cc-archive session history presence
        - config YAML naming and stage prefixes
        - Script-result alignment (task folders, run-result pairing)
        - Code sync: config FnClass/ModelClass → code/haifn/ + code/hainn/ resolution,
          YAML arg signatures, builder ↔ generated file freshness,
          script imports → codebase class existence

      What it generates (write access to docs/ and scripts/ only):
        - docs/TODO.md              created from project scan if missing; updated if exists
        - docs/data-map.md          pipeline flow diagram + stage table (always regenerated)
        - docs/dependency-report.md Fn/model deps, cross-project reuse, missing impls
        - scripts/INDEX.md          built from existing scripts if missing; synced if exists

      Never touches: config/, code/, code-dev/, scripts/*.py, cc-archive/

  /haipipe-project summarize
  /haipipe-project summarize examples/ProjC-Model-WeightPredict
      Generate a plain-English post-development summary. Writes
      docs/project-summary.md with a short description, key results table,
      and an ASCII pipeline flow chart. Updates scripts/INDEX.md statuses.
      Designed to be readable by someone with zero prior context.

  /haipipe-project organize
  /haipipe-project organize examples/ProjC-Model-WeightPredict
      Inventory all project files (and related external Fn/model files),
      propose a reorganization to match the standard four-part layout,
      and — if the user approves — apply moves then immediately verify
      that all imports and paths still work.

  /haipipe-project organize verify
  /haipipe-project organize verify examples/ProjC-Model-WeightPredict
      Skip inventory and proposal; run only the post-reorganization
      verification (import resolution, config FnClass check, relative paths).
      Use after manual file moves.

  /haipipe-project nb
  /haipipe-project nb examples/ProjC-Model-WeightPredict
      Create a demo notebook for a pipeline stage segment. Writes a cell-wise
      .py script (source of truth) then converts to .ipynb. Manages nb/INDEX.md
      and docs/nb-plan.md. Follows the notebook-cell-python skill conventions.

  /haipipe-project help [question]
      Describe what you want in plain English. Routes your request to the
      right subskill and specific step, and suggests the exact call to make.
      Example: /haipipe-project help "I want to see what files are in this project"

What It Creates / Generates
-----------------------------

New projects (/haipipe-project new):

  examples/Proj{Series}-{Category}-{Num}-{Name}/
  +-- cc-archive/        CC session history (cc_*.md, di_*.md)
  +-- config/            YAML pipeline configs per stage
  +-- scripts/           Task-folder layout: scripts/{task}/ contains {task}.py + runs/ + results/
  |   +-- INDEX.md           Mandatory task index (task, data, stage, description, status)
  +-- docs/              Planning and summary documents
  |   +-- TODO.md            Pre-filled pipeline progress tracker
  |   +-- project-summary.md Generated by /haipipe-project summarize
  +-- nb/                Demo notebooks (optional; .py source -> .ipynb converted)
      +-- INDEX.md           Notebook coverage index

Existing projects (/haipipe-project review):

  Generates all docs/ files and scripts/INDEX.md from what already exists.
  No restructuring — only adds what is missing.
      +-- docs/data-map.md          Pipeline flow diagram (from config/ YAMLs)
      +-- docs/dependency-report.md Fn/model deps + cross-project reuse

Two shared indexes across the codebase:

  code/INDEX.md        All implemented Fns and models — checked before
                       creating anything new (reuse across projects first)
  scripts/INDEX.md     Per-project script index by data + functionality

Write Access Summary
---------------------

  Command           Writes to                           Never touches
  ----------------  ----------------------------------  -------------------------
  new               examples/{id}/, code-dev/,          (creates new files only)
                    code/hainn/, code/INDEX.md
  review            docs/, scripts/INDEX.md             config/, code/, code-dev/
  summarize         docs/project-summary.md,            config/, code/, code-dev/
                    scripts/INDEX.md (status only)
  organize          docs/organize-report.md,            config/, code/, code-dev/,
                    examples/{id}/ (moves only),        cc-archive/ session files
                    scripts/INDEX.md (if renamed)
  organize verify   docs/organize-report.md (append)   (read only — no moves)
  nb                nb/ (.py + .ipynb), nb/INDEX.md,   config/, code/, scripts/
                    docs/nb-plan.md
  help              (none — read only, routing only)    (everything)
