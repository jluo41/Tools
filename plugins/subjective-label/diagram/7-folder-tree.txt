📂  Folder Trees — plugin source · project artifacts
=====================================================

  Two trees: the plugin code (this repo) and the {project_dir}/
  the user runs commands against. No agent ever writes to the
  former; everything lands in the latter.

─§ 📁 {project_dir}/ — what each run builds ────────────────

  📂 {project_dir}/
  │
  ├── config.yaml                        topic · schema · model prefs
  ├── .state.json                        🤖 moderator-owned
  │
  ├── sample/
  │   └── sample.jsonl                   📥 raw input texts
  │
  ├── gallery/                           📚 gallery-keeper writes
  │   ├── gallery.json                   curated labeled examples
  │   ├── guideline.md                   prompt-ready rules
  │   ├── panel_config.json              this iter's persona mix
  │   └── history/
  │       ├── iter_1.diff
  │       ├── iter_2.diff
  │       └── CHANGELOG.md
  │
  ├── iterations/                        🔁 one folder per iter
  │   └── iter_N/
  │       ├── candidate_pool.jsonl       ← 📥 sampler (~100 items)
  │       ├── batch.jsonl                ← 🔍 prober (20-30 items)
  │       ├── panel_labels.jsonl         ← 👥 panel
  │       ├── panel_kappa.json           ← 👥 panel
  │       ├── disagreements.md           ← ⚖️ analyzer (narrative)
  │       ├── disagreement_items.jsonl   ← ⚖️ analyzer (per-item)
  │       └── researcher_decisions.jsonl ← 🤖 moderator (post-dialogue)
  │
  ├── cache/                             ⚙️ regenerable
  │   ├── embeddings/                    📐 embedder
  │   │   ├── vectors/{sha1}.npy
  │   │   ├── manifest.jsonl
  │   │   ├── gallery_index.faiss
  │   │   └── gallery_index.meta.json
  │   ├── classifier/                    🧠 classifier (per-iter)
  │   │   ├── iter_N/
  │   │   │   ├── model.pkl
  │   │   │   └── train_metrics.json
  │   │   └── latest -> iter_N/
  │   └── sampler/
  │       ├── init_map.jsonl
  │       └── diagnostic_{ts}.jsonl
  │
  ├── validation/                        📊 validator writes
  │   ├── trajectory.jsonl               κ over iterations
  │   ├── {dataset}_iter{N}_report.md
  │   ├── {dataset}_heldout_sample.jsonl
  │   └── _cache/{dataset}/              HF dataset cache
  │
  └── output/                            🎯 from /sl-scale
      ├── preflight_sample.jsonl         500-item cost estimate
      ├── annotations.jsonl              final labels
      ├── human_review_queue.jsonl       Tier 2 support < 0.6
      └── scale_report.md                cost · throughput · tier mix


─§ 🔑 Boundary rule ────────────────────────────────────────

  🚫 No agent writes to plugin source files.
  💾 Every artifact lives under {project_dir}/.
  🧾 The gallery + guideline are the canonical output;
     iterations/cache/validation are audit + regeneration scaffolding.
