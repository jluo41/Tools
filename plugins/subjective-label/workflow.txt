🧪 subjective-label plugin — v0.3.0
====================================

Three nested loops · 9 agents · 5 personas · 1 researcher.


──────────────────────────────────────────────────────────────────────────
🔁 THE THREE LOOPS (at a glance)
──────────────────────────────────────────────────────────────────────────

BIG LOOP — project lifecycle

  +----------+     +--------------+     +---------------+     +-----------+
  | 🚀 init  |---->| 🔁 iterate   |---->| 📊 validate   |---->| 🎯 scale  |
  +----------+     | (run N times)|     | (vs human κ)  |     +-----------+
                   +--------------+     +---------------+
                          ▲                      │
                          └── 🚩 STALLED ────────┘


MEDIUM LOOP — inside each /sl-iterate (runs 3-6 times per project)

  📥 Sample → 🔍 Prober → 👥 Panel → ⚖️ Analyzer → 🧑 Researcher
                                                        │
                                                        ▼
                                       📚 Gallery ← 🧠 Classifier (train)


SMALL LOOP — 3-tier cascade inside /sl-scale (runs per item, 100K-10M items)

  Tier 0 → Tier 1 → Tier 2
   📐       🧠       👥
  k-NN    logreg   LLM Panel
  ~70%    ~20%     ~10%  ← typical distribution



──────────────────────────────────────────────────────────────────────────
👥 AGENT ROSTER (9 workers)
──────────────────────────────────────────────────────────────────────────

+-------------------------+----------------------------------------+
| 🤖 Agent                | Role                                   |
+-------------------------+----------------------------------------+
| 🤖 moderator            | State machine · only talks to human   |
| 🤖 sampler              | Stage-aware item selection            |
| 🤖 embedder             | SBERT + FAISS · speed tool            |
| 🤖 classifier           | Trained small model · hard-mining     |
| 🤖 prober               | LLM-judgment layer on candidates      |
| 🤖 labeler-panel        | Spawns 3-5 personas in parallel       |
| 🤖 disagreement-analyzer| Sorts disagreement A / B / C / D      |
| 🤖 gallery-keeper       | Sole writer of gallery + guideline    |
| 🤖 validator            | Public dataset κ vs human ceiling     |
+-------------------------+----------------------------------------+

+-------------------+----------------------------------------+
| 🎭 Persona        | Lens                                   |
+-------------------+----------------------------------------+
| 🎭 close-reader   | Slow · evidence-based · cites phrases |
| 🎭 plain-reader   | Ordinary reader intuition             |
| 🎭 skeptic        | Adversarial · finds counter-readings  |
| 🎭 fast-patterns  | Shallow · high-throughput             |
| 🎭 domain-expert  | Parametrized by topic domain          |
+-------------------+----------------------------------------+

──────────────────────────────────────────────────────────────────────────
🎯 KEY IDEAS IN ONE LINE EACH
──────────────────────────────────────────────────────────────────────────

  1. 🧑 Researcher talks ONLY to 🤖 moderator.
  2. 📥 Sampler is the single entry point for any "pick N items" decision.
  3. 📐 Embedder = speed tool. 👥 Panel = judgment tool. Never swap these.
  4. 🧠 Classifier trained after every iteration → drives next-iter hard-mining
     AND serves as Tier 1 of the scale cascade.
  5. ⚖️ Disagreement is signal, not noise (Plank 2022) —
     Analyzer turns disagreement into 5-10 researcher decisions per iteration.
  6. 📊 Convergence = agent-panel κ matches public-dataset human κ ceiling.
  7. 🎯 3-tier cascade: ~70% items free, ~20% cheap, ~10% expensive.
