─§ 🌿 Journal (Stack A) write band — deliberate ────────────

  Mindset: every change is a researcher decision. No auto-loop.

  +---------------------------+---------------------------+
  | Skill                     | Purpose                   |
  +---------------------------+---------------------------+
  | 06 scientific-writing     | Draft / rewrite a section |
  |                           | with reverse outline      |
  | 06 writing-systems-papers | Systems-paper template    |
  |                           | (10-12 pages, OSDI/SOSP/  |
  |                           | ASPLOS-shaped)            |
  | 07 manuscript-optimizer   | Fix claim structure +     |
  |                           | evidence chain            |
  | 07 results-section-       | Local Results flow when   |
  |    revision               | claims are stable         |
  | 07 paper-revise           | Section-by-section revise |
  | 08 paper-compile          | Build PDF · fix errors    |
  +---------------------------+---------------------------+

  ⛔ At Stack A, do NOT auto-paper-improvement-loop —
     it's a conference flow, not a journal flow.

─§ 📚 Refresh-notes discipline (required before 06) ────────

  Before any heavy writing/revision, refresh:

    📄 notes/project_truth.md       what the paper currently claims
    📄 notes/result_summary.md      locked · directional · gapped
    📄 notes/paper_handoff.md       ready to draft vs blocked
    📄 notes/claim_evidence_map.md  claim → evidence → status
    📄 notes/decision_log.md        cross-session decisions

  paper-bootstrap creates these. Manuscript-optimizer reads them.

  ⚠️ Running revision skills against stale notes produces
     confident revisions of yesterday's claims. (PRINCIPLES #10)
