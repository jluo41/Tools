🚂  Workflows — 9 Cross-Phase Orchestrators
============================================

  `skills/_workflows/*` are the entry points: each one chains
  multiple phase-skills with parameter pass-through. Underscore
  prefix sorts them first under `ls`.


─§ 📜 Roster ───────────────────────────────────────────────

  +----+-------------------------+----------------------------------+
  |    | Workflow                | Role                             |
  +----+-------------------------+----------------------------------+
  | 🚂 | research-pipeline       | Full W1→W4 (idea → submission)   |
  | 🌿 | paper-workflow          | Top-level journal router         |
  | 🤖 | paper-writing (W3)      | Conference paper auto-pipeline   |
  | 🔭 | idea-discovery (W1)     | Idea exploration pipeline        |
  | 🤖 | idea-discovery-robot    | W1 specialized for robotics/EAI  |
  | 🌉 | experiment-bridge (W1.5)| Plan → running code + log        |
  | 🔄 | (auto-review-loop)*     | W2 — review loop (atomic skill)  |
  | ✉️ | rebuttal (W4)           | Reviews → safe text rebuttal     |
  | 📖 | is-paper-workflow       | IS-journal end-to-end            |
  | 🔒 | patent-pipeline         | Invention → filing               |
  +----+-------------------------+----------------------------------+

  * auto-review-loop lives in 10_review/, not _workflows/, but
    plays the W2 role in the canonical W1→W4 chain.

─§ 🌉 Workflow chaining — common parameters ────────────────

  All workflows accept and pass through:

    ─ effort:        🪶 lite | balanced | max | beast
    ─ human checkpoint:  true | false
    ─ AUTO_PROCEED:      true | false
    ─ difficulty:    medium | hard | nightmare       (W2 reviewer)
    ─ venue:         ICLR | NeurIPS | ICML | …       (W1.5+)
    ─ sources:       web · zotero · deepxiv · …      (W1)
    ─ gpu:           local · remote · vast · modal   (W1.5+)
