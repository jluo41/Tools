─§ 🔀 Loop-first vs audit-first ────────────────────────────

  STACK B (conference)              STACK A (journal)
  loop-first 🤖                     audit-first 🌿
  ─────────────────                 ──────────────────

   draft                              draft
     │                                  │
     ▼                                  ▼
   auto-review-loop ⟲                citation-verifier
     │                                  │
     │  iterate                         ▼
     │                                submission-audit  🚪
     ▼                                  │
   paper-claim-audit 🚪                 │  green?
     │  green @ max|beast            ┌──┴──┐
     ▼                              ✅      ❌
   submission-audit 🚪               │      │
     │                                ▼      ▼
     ▼                            submit  fix claim
   📤 submit                                 │
                                              ▼
                                         iterate

  📌 Why the difference?
     Conference: many comparisons missing → loop catches them
                  fast; final audits gate at max|beast.
     Journal:    weak claim → loop will polish weakness;
                  audit MUST come first so the loop never runs
                  on shaky evidence (PRINCIPLES #7).

─§ 🤖 Cross-model protocol ─────────────────────────────────

  AGENT_GUIDE.md defines the executor/reviewer split:

      🛠️ Executor   (Claude / Codex)
                   writes code · runs experiments · drafts paper
                                │
                                │  output → file
                                ▼
      🔍 Reviewer   (GPT-5.4 / Gemini / GLM)
                   critiques · scores · demands revisions

  Rules:
    1. Different model FAMILIES (no Claude reviews Claude).
    2. Reviewer independence — pass file paths, never summaries.
    3. Executor must NOT judge its own eval code — reviewer
       audits it directly (experiment-integrity).

  This split is what makes auto-review-loop's gating credible.

─§ 🛡️ Anti-principles (what review will NEVER do) ─────────

  ⛔ "Make my paper better" one-shot button —
     every loop gates on claims being locked first.
  ⛔ Polishing without a claim-evidence map.
  ⛔ Auto-generated citations without verification.
  ⛔ Silent skill substitution (loop-where-audit-belongs).
