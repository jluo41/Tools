# Use /diagram-ascii when explaining choices or current status

When explaining a choice (e.g. which claim is primary, why this venue, what
the claim hierarchy looks like) or summarizing the current paper status, use
/diagram-ascii to produce a visual diagram rather than relying on prose or
plain tables alone. The user understands the paper's state much faster from
a visual diagram than from text.

Applies to:
- Claim architecture and venue coupling (hierarchy + evidence map)
- Lifecycle status and frontier explanation
- Display inventory and evidence routing
- Any "here's what we have and what's next" summary
- Any "here's why we chose X" explanation

How: invoke /diagram-ascii inline before or alongside the prose explanation.
The diagram IS the explanation; prose supplements it.
