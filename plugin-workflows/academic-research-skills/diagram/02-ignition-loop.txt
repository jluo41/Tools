🔁  Ignition Loop — One-by-One Narrative x Probe
================================================

  This diagram extends 01-paper-ignition.txt.

  The first insight was:

      🧪 Probe Line + 🎭 Narrative Line → 🔥 Paper Ignition

  The sharper insight is:

      They should not run as two completed phases.
      They should alternate node by node.

  Narrative should shape the next probe.
  Probe results should reshape the next narrative.


─§ ❌ Bad sequence ────────────────────────────────────────

  These two workflows both fail:

  A. Narrative first, probe later

      🎭 beautiful story
          │
          ▼
      🧪 probe tries to support it
          │
          ▼
      ❌ risk: story becomes wishful overclaim


  B. Probe first, narrative later

      🧪 endless probes
          │
          ▼
      🎭 story added after the fact
          │
          ▼
      ❌ risk: paper becomes a dull result dump


  Both treat one line as primary and the other as decoration.
  A real paper needs both to discipline each other.


─§ ✅ Good sequence: one by one ───────────────────────────

  The unit is not "whole narrative" or "whole experiment".
  The unit is a small decision node.

      🎭 Narrative Node 1
          What story might be true?
          What claim would matter?
          What would ignite the author?
              │
              ▼
      🧪 Probe Node 1
          What can we test now?
          What result would support / kill the story?
              │
              ▼
      🔥 Judgment Node 1
          Did this make the story more alive?
          Did it clarify the next move?
              │
      ┌───────┴────────────────────────────┐
      ▼                                    ▼
  🎭 revise narrative                  🧪 run next experiment
      │                                    │
      └────────────────┬───────────────────┘
                       ▼
              🔁 next ignition loop


─§ 🧭 Minimal loop ────────────────────────────────────────

  +----------------------+       +----------------------+
  | 🎭 Narrative          |       | 🧪 Probe              |
  |----------------------|       |----------------------|
  | angle                |──────▶| test                 |
  | claim candidate      |       | result               |
  | reader urgency       |◀──────| evidence boundary    |
  | next question        |       | surprise / failure   |
  +----------------------+       +----------------------+
              │                              │
              └──────────┬───────────────────┘
                         ▼
                🔥 Ignition Judgment
             "Am I more eager to continue?"


─§ 🔥 The judgment question ───────────────────────────────

  Every node asks the same question:

      Did this ignite the paper?

  More precise:

  +----------------------+-----------------------------------+
  | Test                 | What it means                     |
  +----------------------+-----------------------------------+
  | 🧠 Clarity            | Do I know what this means now?    |
  | 📊 Support            | Is the claim more honest now?     |
  | ⚡ Energy             | Do I feel more eager to sell it?  |
  | 🧭 Direction          | Do I know the next move?          |
  | 🚫 Boundary           | Do I know what NOT to claim?      |
  +----------------------+-----------------------------------+

  If the answer is no, do not draft.
  Either rethink the story or run a sharper experiment.


─§ 🌳 Node decision tree ──────────────────────────────────

                     🔥 judgment after a node
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
      🟢 ignited          🟡 partial          🔴 not ignited
          │                   │                   │
          ▼                   ▼                   ▼
   act immediately      refine next node       stop / reframe
   while energy is hot  with one question      before wasting work
          │                   │                   │
          ▼                   ▼                   ▼
      🧪 run probe         🎭 adjust narrative    🧭 find better angle
   or ✍️ draft section  or 🧪 run small test    or abandon weak route


─§ 🎭 Narrative before action ─────────────────────────────

  Before any action, make the narrative intent explicit:

      "I am running this experiment because if X happens,
       it will let me claim Y, which matters because Z."

  Without this sentence, the experiment is blind labor.

  After the action, update the sentence:

      "The result supports / weakens / changes Y.
       Therefore the next story move is..."


─§ 🧪 Probe before narrative claim ────────────────────────

  Before any strong narrative claim, ask:

      "What result earns this sentence?"

  If no result earns it, the sentence is not a claim yet.
  It is a hope, a hunch, or a motivation.

  That is allowed, but it must be labeled honestly.


─§ 📌 One-for-one rule ────────────────────────────────────

  One narrative move should have one probe pressure.
  One probe result should have one narrative interpretation.

  +----------------------------+-------------------------------+
  | Move                       | Pair it with                  |
  +----------------------------+-------------------------------+
  | 🎭 "This is the gap"        | 🧪 evidence that gap exists    |
  | 🎭 "This mechanism matters" | 🧪 test isolating mechanism    |
  | 🎭 "This result is new"     | 🧪 baseline / prior comparison |
  | 🧪 surprising result        | 🎭 revised story angle         |
  | 🧪 null result              | 🎭 boundary / limitation story |
  +----------------------------+-------------------------------+


─§ ⏱️ Act at ignition moment ──────────────────────────────

  The key is not just to be inspired.
  The key is to know exactly WHEN inspiration becomes actionable.

      When the story becomes clear enough,
      and the next experiment becomes obvious,
      act immediately.

  Good workflow preserves the hot moment:

      🔥 "I see it now"
          │
          ├─▶ write the narrative contract
          ├─▶ define the probe contract
          ├─▶ run the next test
          └─▶ capture why this matters before it cools


─§ 🧩 Implication for ARS ─────────────────────────────────

  Current ARS is mostly stage-linear:

      1 RESEARCH → 2 WRITE → 2.5 INTEGRITY → 3 REVIEW

  Better mental model:

      🔁 micro-loop until ignition:

          🎭 narrative node
              ↕
          🧪 probe node
              ↕
          🔥 judgment node

      Then:

          ✍️ write only the part that is ignited

  This suggests ARS needs a pre-writing conductor that can say:

      "Do not draft yet."
      "Run this small probe next."
      "This result changed the story."
      "This section is ready to write now."


─§ 💬 Quotes to preserve ──────────────────────────────────

  "Narrative and probe are not two phases.
   They are alternating judgments."

  "Before action, know the story reason for the action.
   After action, update the story with what reality answered."

  "Every probe asks: does this make the story more honest?
   Every narrative asks: what probe would make this real?"

  "The standard is not whether a task is done.
   The standard is whether the paper is more ignited."

  "When you know the exact moment you were ignited,
   capture it and move before it cools."


─§ 🧨 Final memory ────────────────────────────────────────

  Do not finish narrative, then run probes.
  Do not finish probes, then invent narrative.

  Alternate them.

  One story move.
  One probe move.
  One ignition judgment.

  Repeat until the paper wants to be written.
